# Hi-C based Modeling of Chromatin Structure Energy Landscape During Cell Fate Determination using an Ising–Simulated Annealing Framework

This repository contains Python and R code used to reproduce the analyses and figures presented in the study above.

The pipeline implements a full computational workflow for single-cell Hi-C data analysis, ranging from raw contact matrix processing to energy landscape reconstruction of chromatin state dynamics.

---

## Repository Structure, Pipeline Overview

### scripts/

All downstream analyses require **normalized single-cell Hi-C (scHi-C) contact matrices**. Therefore, before reproducing any analyses or figures, you should complete the preprocessing and normalization pipeline in the order described below.

Run the scripts in the `scripts/` directory in the following sequence:

1. **Preprocessing**
2. **scHi-C Normalization**
3. **Compartment Score and TAD Detection**

   After normalization, the following two pipelines can be run **independently**:

   * `scripts/sc-compartment-score/`
   * `scripts/TAD-detection/`

4. **Ising Energy**
5. **Simulated Annealing Monte Carlo**
     


For detailed instructions, required inputs, configurable parameters, and output files for each stage, please refer to the corresponding `README.md` file in each pipeline directory.
---

### reproducing_figures/

Jupyter notebooks for figure reproduction.

Each notebook corresponds to a specific figure (or figure panel) in the manuscript and provides step-by-step instructions to regenerate results from processed outputs generated by the scripts pipeline.

- **Reference paper:**  
  Droplet Hi-C enables scalable, single-cell profiling of chromatin architecture in heterogeneous tissues ([DOI: 10.1038/s41587-024-02447-1](https://www.nature.com/articles/s41587-024-02447-1))

Raw data are not included in this repository.  
Users must download the dataset from GEO and place it in the appropriate `data/` directory before running the pipeline.
